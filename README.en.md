# Practical Workplace English for Chinese Developers

This repository is a practical playbook for Chinese software engineers.  
It is not an exam track and not a vocabulary-count game. The goal is simple: help you communicate clearly at work in English.

Chinese readers can use: [中文 README](README.md)

## What this repository helps with

- Build usable speaking and writing ability for real engineering work.
- Improve English in daily scenarios: documentation, code review, reporting, and collaboration.
- Use a repeatable loop instead of random practice: `repeat -> shadowing -> write -> review -> refine`.

## Common struggles this playbook targets

- You can read some English but freeze when speaking or writing.
- You do lots of listening practice but still miss technical content.
- You memorize words but cannot use them in real communication.
- You worry about Chinglish and avoid writing.
- You are unsure what to practice daily and how to measure progress.

## Core ideas

- Speaking and writing are the main levers.
- Listening improves through speaking; grammar improves through writing with feedback.
- Learn in context, not by isolated memorization.
- Do not chase perfection first; chase stable, understandable output.
